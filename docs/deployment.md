# Развёртывание LaTeX Editor ЛЭТИ

## Требования к серверу

- Linux (Ubuntu 22.04 LTS или аналог)
- Node.js 18+
- TeX Live (pdflatex в PATH)
- Nginx (reverse proxy, опционально SSL)

## Установка

1. Клонировать репозиторий в `/opt/leti-latex-editor` (или другой каталог).
2. Установить зависимости: `npm install`, затем `npm run build` (сборка Tailwind в `public/css/style.css`).
3. Создать каталог данных: `mkdir -p data/projects`.
4. Скопировать `deploy/systemd/leti-latex-editor.service` в `/etc/systemd/system/`, при необходимости изменить `WorkingDirectory` и `User`/`Group`.
5. Создать файл окружения `/etc/leti-latex-editor.env` (опционально):
   - `DATA_DIR` — путь к каталогу данных (по умолчанию `{base_dir}/data`)
   - `MAX_FILE_SIZE_BYTES` — лимит размера файла (по умолчанию 1048576)
   - `COMPILE_TIMEOUT_SECONDS` — таймаут компиляции (по умолчанию 60)
   - `SECRET_KEY` — секрет для сессий (обязательно сменить в production)
   - `MAX_PROJECTS_PER_USER` — лимит проектов на пользователя (по умолчанию 50)
   - `PORT` — порт приложения (по умолчанию 8000)
6. Запуск: `systemctl enable leti-latex-editor`, `systemctl start leti-latex-editor`.
7. Nginx: скопировать `deploy/nginx/leti-latex-editor.conf` в `/etc/nginx/sites-available/`, включить сайт, настроить SSL при необходимости (Let's Encrypt или корпоративный сертификат).

## Резервное копирование

- Регулярно копировать каталог `data/` (включая `data/projects/` и при наличии `data/users.json`).
- Рекомендуется cron-задача (например, ежедневно) с rsync или архивом на отдельный носитель.
- Восстановление: остановить сервис, восстановить `data/`, запустить сервис.

## Проверка после деплоя

После выкладки приложения рекомендуется проверить сценарии:

1. **Главная и вход:** открыть корень сайта — отображается лендинг (заголовок «Редактор LaTeX ЛЭТИ», блок возможностей, шапка и подвал). Кнопка «Войти» ведёт на экран входа (`#/login`). Ввод учётных данных и успешный вход перенаправляют в личный кабинет (`#/account`).
2. **Личный кабинет:** в ЛК отображаются список проектов (карточки), кнопка «Новый проект», ссылка «В редактор». Создание нового проекта открывает редактор с выбранным проектом.
3. **Редактор:** переход по «Редактор» или «Открыть в редакторе» — загрузка списка проектов, выбор проекта, открытие файлов, компиляция. Проверить: сохранение файла, кнопка «Скомпилировать», превью PDF, лог компиляции и переход по ошибкам.
4. **Навигация и выход:** из шапки доступны «Главная», «Личный кабинет», «Редактор»; кнопка «Выход» возвращает на главную с экраном входа при повторном заходе в защищённые разделы.
5. **Элементы управления:** кнопки (первичные, вторичные), поля ввода, чекбокс «Авто-компиляция», контекстное меню по ПКМ по файлу, модальные окна (история, новый проект) — все реагируют на hover/focus, закрытие по Escape и клику вне.

При развёртывании под префиксом (например `https://latex.etu.ru/app/`) задать базовый путь API в конфигурации фронтенда (например в `public/js/api.js`), чтобы запросы уходили на `/app/api/...`.

## Обновление приложения

- Остановить сервис: `systemctl stop leti-latex-editor`.
- Обновить код (git pull или копирование файлов).
- Обновить зависимости: `npm install`, при изменении стилей — `npm run build`.
- Запустить: `systemctl start leti-latex-editor`.
